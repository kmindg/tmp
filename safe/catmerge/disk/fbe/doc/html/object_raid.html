Raid Group Non-paged Metadata

The raid group's non-paged metadata contains the following fields:

  • needs rebuild element[2] — information on a position that has chunks requiring rebuild.
    • checkpoint — lowest chunk that requires a rebuild.
    • position — position in the array this nr element corresponds to.
  • verify_for_user_checkpoint — lowest chunk that requires a user requested verify.
  • verify_for_error_checkpoint — lowest chunk that requires a verify because of I/O errors.
  • verify_for_abort_checkpoint — lowest chunk that requires a verify because of aborted I/Os.
  • verify_for_read_only_checkpoint — lowest chunk that requires a verify because of a read only verify.
  • need_rebuild_on_demand[16] — drives that require rebuild logging (one bit per drive in the raid group).
  • need_rebuild_all_drive[16] — drives that require all of the drive to be rebuilt (one bit per drive in the raid group).
  • paged_metadata_metadata[32] — metadata bit sequences that cover the paged metadata.

Raid Group Paged Metadata

The raid group's paged metadata contains the following bits:

  • verify_for_user — the corresponding user data chunk needs to be verified by user request.
  • verify_for_error_bit — the corresponding user data chunk needs to be verified because of I/O errors.
  • verify_for_abort — the corresponding user data chunk needs to be verified because of I/O aborts.
  • verify_for_read_only — the corresponding user data chunk needs to be read only verified.
  • reserved_bits[12] — unused bits, reserved for future use.
  • need_rebuild_bits[16] — the corresponding user data chunk needs to be rebuilt (one bit per drive in the raid group).
  • reserved_bits[32] — unused bits, reserved for future use.
