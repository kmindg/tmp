Virtual Drive Object

The virtual drive object is responsible for managing on-line drive sparing. On-line drive sparing can occur for the following use cases:

  • A physical drive fails or is removed. In this case the system provides a hot spare if a suitable drive is available.
  • The physical drive error thresholds rise to a level that indicates that the drive is nearing its end-of-life. In this case the system may automatically replace the dying drive with a hot spare.
  • The user requests that a drive be proactively replaced.
  • A permanent spare is provided to replace a hot spare.

Virtual Drive Control Path

The virtual drive control path performs the following operations:

  • Hot Spare
  • Equalize
  • Proactive Copy

Virtual Drive Non-paged Metadata

Because the virtual drive is derived from the raid group class, its non-paged metadata is the raid group's non-paged metadata.

Virtual Drive Paged Metadata

As with the non-paged metadata the virtual drive paged metadata is the raid group's paged metadata.
