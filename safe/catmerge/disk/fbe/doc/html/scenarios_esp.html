Environment Services Package Scenarios

The ESP scenarios are...

Flare Re-work Scenarios

  • BobTheDinosaur: flexport interface to Physical Package.
  • Catbert: enclosure firmware download using the FBE API interface.
  • Dilbert: event logging infrastructure for Flare.
  • Dogbert: asynchronous notification for physical component state changes.
    This test the ability to register for enclosure state changes and to handle them in ESP.
  • Orko: handling of physical component state changes.
  • PenrodRooch: removal of all unused modules from Flare.
  • Ratbert: removal of all diplex interface to FBE API and EDAL calls.
  • Spiderman: Environment Service package framework. This test validates that all the required ESP services are started and that the FBE API is initialized correctly.
  • Wimpy: external client (e.g. BIOS, POST, Flex Ports), read/write access to persistent storage area.
    This test validates the ability to read and write the persistent storage area for external clients; e.g., NDU to store BIOS and POST Firmware Image.
  • edward_and_bella: Single SP power supply firmware upgrade.
    This test tests Single SP power supply firmware upgrade state transition and queuing logic.
  • curious_george: Power supply firmware upgrade status reporting.
    This test verifies the initiation of power supply firmware upgrade when the power supply is added and the upgrade status reporting in various scenarios.
  • vavoom: Single SP enclosure firmware upgrade.
    This test tests Single SP enclsoure firmware upgrade state transition and queuing logic.
  • poindexter: LCC firmware upgrade status reporting.
    This test verifies the initiation of LCC firmware upgrade when the LCC is added and the upgrade status reporting in various scenarios.
  • edward_the_scissorhands: LCC firmware upgrade status reporting.
    This test verifies firmware upgrade abort and resume.
  • grandma_wolf: Firmware upgrade error handling.
    This test verifies firmware upgrade error handling.
  • marcus: XPE and DPE handling.
    This test verifies the XPE and DPE attributes.
  • tintin: Multiple devices firmware upgrade coordination and prioritization.
    This test tests multiple devices firmware upgrade coordination and prioritization.
  • siren: Verify the initialization of processor enclosure PS mode.
    This test verifies the initialization of processor enclosure PS mode.
  • grandma_robot Single SP Fan firmware upgrade.
    This test tests Single SP Fan firmware upgrade state transition and queuing logic.
  • tarzan The voyager enclosure removal and addition.
    This test tests the voyager enclosure removal and addition.
  • dumbo The Ancho 12G enclosure LCC firmware upgrade.
    This test tests Ancho 12G DAE LCC firmware upgrade state transition and queuing logic.
  • johnny_tractor The Cayenne 12G enclosure LCC and PS firmware upgrade.
    This test tests Cayenne 12G enclosure LCC and PS firmware upgrade state transition and queuing logic.
  • allie_gator The Naga 12G enclosure LCC and PS firmware upgrade.
    This test tests Naga 12G enclosure LCC and PS firmware upgrade state transition and queuing logic.
  • corey_combine The Tabasco 12G enclosure LCC firmware upgrade.
    This test tests Tabasco 12G DAE LCC firmware upgrade state transition and queuing logic.
  • wall_e The Oberon DPE SP firmware upgrade.
    This test tests Oberon DPE SP firmware upgrade state transition and queuing logic.
  • peter_rabbit The Hyperion DPE Base Module firmware upgrade.
    This test tests Hyperion DPE Base Module firmware upgrade state transition and queuing logic.

Admin Interface Scenarios

  • HaroldHamGravy: framework support for interface to Navi.
  • SeaHag: returning configuration information to Navi.
  • EugeneTheJeep: issuing configuration set commands from Navi (e.g. create a RG).
  • BarnacleBill: issuing configuration change commands from Navi (e.g. modify a RG attribute).
